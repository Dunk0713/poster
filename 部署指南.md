# 小红书笔记生成器 · 上线部署指南（GitHub Pages）

> 面向非技术用户。全程网页点点点即可，不用装任何软件。
> 最终你会得到一个网址（形如 `https://你的用户名.github.io/仓库名/`），任何人打开即可使用。

---

## 一、准备：注册 GitHub 账号（已有可跳过）

1. 打开 https://github.com → 点 **Sign up** 注册（邮箱 + 密码 + 用户名）。
2. 按邮件验证完成即可。用户名建议用英文（会出现在网址里）。

## 二、新建一个空仓库（Repository）

1. 登录后，右上角 **+** → **New repository**。
2. **Repository name** 随便起个英文名，例如 `rednote-card`（这会成为网址的一部分）。
3. 选 **Public**（公开，别人才能访问；GitHub Pages 免费版需公开）。
4. **不要**勾选 “Add a README”等任何初始化选项（保持空仓库）。
5. 点 **Create repository**。
6. 创建后页面会显示仓库地址，形如：
   `https://github.com/你的用户名/rednote-card.git`
   —— **把这个地址复制好，发给我**，我来把代码推上去（这一步交给我，你不用敲命令）。

## 三、我推送代码后，你来开启 Pages（关键 3 步）

代码推送成功后，在你的仓库页面：

1. 点上方 **Settings**（设置）。
2. 左侧菜单点 **Pages**。
3. **Build and deployment → Source** 选 **Deploy from a branch**；
   下面 **Branch** 选 **`main`**，文件夹选 **`/docs`** → 点 **Save**。
4. 等 1–3 分钟刷新本页，顶部会出现绿色提示和网址：
   `Your site is live at https://你的用户名.github.io/rednote-card/`
   —— 这就是上线网址，发给任何人都能用。

> 注意网址结尾的 **斜杠 /** 要保留（`.../rednote-card/`），直接能打开。

---

## 四、它能干什么 / 注意事项

- 打开网址即用，**手机/电脑浏览器都行**；做好的笔记点「导出当前页」存成图片发小红书。
- **内容只存在你自己的浏览器里**（自动暂存）。换设备/清缓存会丢，重要内容记得点「**存档**」下载 JSON 备份，下次「**打开**」恢复。
- 全部资源已打包进站点，**国内可直接访问**，不依赖任何会被墙的 CDN。
- 字体：屏幕上「文人/手写」气质用霞鹜文楷；**导出的图片里**衬线/手写字会回退成系统楷体（这是为保证导出又快又稳的取舍，默认及多数气质不受影响）。

## 五、以后怎么更新网站

内容/功能有改动后，告诉我，我会重新生成 `docs/` 并推送；GitHub Pages 几分钟内自动更新，网址不变，用户刷新即见新版。

---

## 备选托管（如暂不想用 GitHub）

- 把项目里的 **`docs/` 整个文件夹** 拖到 https://app.netlify.com/drop 或 Cloudflare Pages，也能立即得到一个网址（国内速度一般，应急可用）。
- 国内更快可选：腾讯云静态网站托管 / Gitee Pages（需实名，步骤类似，把 `docs/` 作为站点根目录）。
